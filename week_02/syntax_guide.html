Constructors

This diagram shows the syntax for declaring constructor:

Constructors are called when an object is created to provide a time to setup and initialise the object.

public class Shape
{
  private int _x;
  private int _y;

  public Shape(int x, int y)
  {
    _x = x;
    _y = y;
  }
}

Constructors can require parameters, the code example aboves' constructor requires two ints.

public class Shape
{
  public string Name = "Shawn";
  private int _x;
  private int _y;

  public void Draw()
  {
    // ...
  }
}

Fields

This diagram shows the syntax for fields:

Just like instance methods are "attached" to each object of a given class, fields are variables associated with a given class (unless they're static). Each object has it's own memory allocation for each instance field.

Having a look at the code below, it has three fields, two of which are instance fields. (static fields are not instance fields).

public class Shape
{
  public static string Name = "Shawn";
  private int _x;
  private int _y;
}

Let's take a look at the public static string name = "Shawn";. * firstly, the public modifier, which means any code which has a reference to it can access and change it. * the static modifier means that the field's memory is shared across all Shape objects. * string is the type of the field. * Lastly, it is assigned the value "Shawn".

Properties

Properties are "accessors" to an objects private fields. They can either just return the value of the field, or perform a calculation/expression.

properties are helpful because they act as accessors to fields. This means they provide ways for us to control how much access they have to a field.

This diagram shows the syntax for properties: 

public class Shape()
{
  private int _x;
  public int X
  {
    get {return _x};
    set { _x = value };
  }
}

The get and set keywords tell us that this property has access to getting and setting the field.

A common pattern is to omit the set, which says that this field is readonly: ~cs public class Shape() { private int _x; public int X { get {return _x}; } } ~

We can also use auto-implemented properties, which are a helpful and quicker way to create a field and property. The following code will have a public X property, and a private field. cs public class Shape() { public int X { get; set; } }

Return statement

Returning is an important part of our methods, it's how we signal that the method as finished doing its job, and if the method is a function, it let's us know which value to return to the caller.

This diagram shows the syntax for returning in our code:

There are a few things to note when returning: * Void methods don't return a value, just use the return statement: ie return;. * You must return a value who's type matches the return type of the method, ie: * You can return any value of the correct type, so you can can return a variable (return name;), literal value (return 3;), or calculation (return x + 50/2;). The expression is evaluated, and then returned. ~cs string GetName() { return "John"; // "John" is a string, good :) }

string GetName() { return 3; // 3 is not a string, won't work :( } ~

Example program

We've covered a lot of new syntax this week, why don't you have a look at the following example program and try and identify all of the pieces of syntax we've spoken about? ~cs using System; using SplashKitSDK;

public class Program { public static void Main() { Window w = new Window(...); StorageTank tnk = new StorageTank(100.0, 27.5, 10, 10, 70, 200); } }

public class StorageTank { private Color _liquidColor = Color.Blue; private readonly double _capacity; private double _level;

private int _x, _y; private int _width, _height;

public StorageTank(double capacity, double startLevel, int x, int y, int width, int height) { _x = x; _y = y; _width = width; _height = height;

_capacity = capacity;
_level = startLevel;

}

public Color LiquidColor { get { return _liquidColor; } set { _liquidColor = value; } }

public double Capacity { get { return _capacity; } }

public double Level { get { return _level; } }

public double PercentFull { get { return _level / _capacity; } }

public void AddLiquid(double amount) { _level = _level + amount; }

public void TakeLiquid(double amount) { _level = _level - amount; }

public void Draw(Window w) { double topY = y + height - height * PercentFull; w.FillRectangle(LiquidColor, x, topY, x + width, y + height); w.DrawRectangle(Color.Black, x, y, x + width, y + height); } } ~

Modifier Keywords Cheat Sheet

  • public: Any class which references the object has access.
  • private: Accessible only within the body of the class in which they are declared.
  • static: Declares a member that belongs to the type itself instead of to a specific object.

There are other modifiers, which we'll take a look at in other weeks. See the msdn reference for further information.
